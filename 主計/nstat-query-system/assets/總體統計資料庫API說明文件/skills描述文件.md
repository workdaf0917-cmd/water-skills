# Skill: 台灣總體統計資料庫查詢系統

## 📋 Purpose

透過台灣主計總處「總體統計資料庫 API」查詢各類官方統計數據，涵蓋國民所得、物價、勞工、金融、人口、教育、衛生等 20 個統計領域，支援年度、季度、月度數據查詢。

## 🎯 Use Cases

1. **經濟研究與分析** - 查詢 GDP、CPI、失業率等關鍵經濟指標
2. **政策評估** - 取得人口、教育、社會福利等領域的統計數據
3. **市場研究** - 分析工商業、進出口、證券市場等商業數據
4. **學術研究** - 取得可信的官方統計資料作為研究基礎
5. **報告撰寫** - 快速取得結構化數據，無需手動網站查詢
6. **數據監控** - 追蹤特定指標的時間序列變化

## 🚫 Non-Goals

- 不提供數據預測或趨勢預估（僅歷史數據）
- 不支援跨表複雜查詢或數據計算
- 不提供數據視覺化（僅返回原始數據）
- 不支援即時數據（依主計總處公布時程）

## 📥 Input

### 必填參數

1. **統計領域**（20 個領域之一）
    - 國民所得統計、家庭收支統計、物價統計、勞工統計、工商業統計
    - 營建與不動產統計、交通統計、進出口統計、金融統計、財政統計
    - 證券統計、景氣指標統計、人口統計、警政與消防統計
    - 社會保險及福利統計、教育統計、衛生統計、環保統計、農業統計、社會保障支出統計
2. **資料表名稱**（如：「每人每月總薪資」、「消費者物價指數」）
3. **時間範圍**
    - 起始時間：YYYY（年）、YYYY-MM（月）、YYYY-QX（季）
    - 結束時間：同上格式
    - 驗證：起始時間 ≤ 結束時間

### 選填參數

1. **資料維度篩選**（依各表定義）
    - 統計項目篩選（如：就業者、失業者）
    - 複分類篩選（如：性別、年齡、教育程度）
    - 預設值：查詢所有維度
2. **資料週期**
    - `A`（年度，預設）
    - `Q`（季度）
    - `M`（月度）

### 輸入範例

```
查詢 2020-2023 年每人每月總薪資
查詢 2023 年 1 月到 12 月的消費者物價指數
取得 2022 年 Q1 到 Q4 的 GDP 數據
查詢勞動力統計，按教育程度分，2021-2023 年
```

## 🔄 Workflow

### Step 1: 理解使用者查詢意圖

- 識別**統計領域**（從自然語言推斷，如「薪資」→勞工統計）
- 提取**資料表關鍵字**（如「消費者物價」、「GDP」）
- 解析**時間範圍**和**週期**（年/季/月）
- 識別**維度篩選需求**（如「按性別」、「按年齡」）

### Step 2: 查找功能代碼

使用內建對照表（基於 PDF 附表一）：

```python
# 範例對照結構
FUNCTION_CODES = {
    "勞工統計": {
        "受僱員工人數": "A046101010",
        "每人每月總薪資": "A046201010",
        "每人每月經常性薪資": "A046301010",
        "受僱員工每人每月工時": "A046401010",
        "勞動力-按教育程度分": "A040101020",
        "勞動力-按年齡分": "A040102010",
        # ... 更多
    },
    "物價統計": {
        "消費者物價基本分類指數": "A030101015",
        "躉售物價基本分類指數": "A030202010",
        # ... 更多
    },
    # ... 其他領域
}
```

**如找不到精確匹配**：

- 使用模糊搜尋找出最接近的 3 個資料表
- 詢問使用者確認

### Step 3: 查詢資料表元資料（metadata）

發送請求取得資料表結構：

```
GET https://nstatdb.dgbas.gov.tw/dgbasAll/webMain.aspx?sys=210&funid=[功能代碼]
```

解析回應以取得：

- 統計項（dimension 1）及其項目
- 複分類（dimension 2, 3, ...）及其項目
- 可用的統計期間範圍
- 資料週期類型（年/季/月）

### Step 4: 構建查詢參數

根據使用者需求和資料表結構，構建篩選運算式：

**語法結構**：

```
資料維度篩選 = 維度1項目編碼.維度2項目編碼.維度3項目編碼.資料週期
```

**範例 1**：查詢「勞動力-按教育程度分」的就業者、失業者，性別合計，所有教育程度，月資料

```
篩選運算式: 1+2.1..M
解釋：
  - 維度1（統計項）: 1+2（就業者+失業者）
  - 維度2（性別）: 1（合計）
  - 維度3（教育程度）: 空白（全部）
  - 週期: M（月資料）
```

**範例 2**：查詢所有維度

```
篩選運算式: ...A
解釋：所有維度留空，A表示年資料
```

### Step 5: 發送 API 請求

**完整 URL 格式**：

```
https://nstatdb.dgbas.gov.tw/dgbasAll/webMain.aspx?sdmx/{功能代碼}/{篩選運算式}&startTime={起始時間}&endTime={結束時間}
```

**實際範例**：

```
https://nstatdb.dgbas.gov.tw/dgbasAll/webMain.aspx?sdmx/A046201010/1.1.A&startTime=2020&endTime=2023
```

**請求設定**：

- Method: GET
- 匿名查詢（無需認證）
- Timeout: 30 秒
- Accept: application/json

### Step 6: 解析 API-JSON 回應

API-JSON 結構：

```json
{
  "meta": {
    "prepared": "查詢時間",
    "sender": "資料來源",
    "id": "唯一識別碼",
    "links": "API URL"
  },
  "data": {
    "dataSets": [{
      "series": {
        "0:0:0": {
          "observations": {
            "0": [數值],
            "1": [數值],
            ...
          }
        }
      }
    }]
  },
  "structure": {
    "dimensions": {
      "series": [
        {
          "keyPosition": 0,
          "id": "統計項",
          "name": "統計項",
          "values": [
            {"id": "1", "name": "項目1"},
            {"id": "2", "name": "項目2"}
          ]
        }
      ],
      "observation": [
        {
          "id": "TIME_PERIOD",
          "name": "統計期",
          "values": [
            {"id": "2020", "name": "109年"},
            {"id": "2021", "name": "110年"}
          ]
        }
      ]
    }
  }
}
```

**提取步驟**：

1. 從 `structure.dimensions.series` 取得維度定義
2. 從 `structure.dimensions.observation` 取得統計期對應
3. 從 `data.dataSets[0].series` 取得各維度組合的數據
4. 將 observations 中的數值與統計期對應

### Step 7: 格式化輸出

- 依使用者偏好選擇輸出格式（表格/結構化文字/JSON）
- 加入資料來源、功能代碼、查詢時間等元資訊
- 對數值進行千分位格式化（如適用）
- 標註單位（千人、新台幣元、%、指數等）

### Step 8: 回傳結果

## ⚙️ Constraints & Policy

### 1. 資料使用規範

- **資料來源**：僅使用主計總處官方 API
- **數據準確性**：以主計總處公布為準，不做修改或調整
- **引用標註**：必須標註「資料來源：行政院主計總處」

### 2. 查詢限制

- **URL 長度上限**：1,000 字元
- **單次查詢時間範圍**：建議 ≤ 20 年（避免數據量過大）
- **維度組合上限**：受 URL 長度限制

### 3. 數據解釋原則

- 僅提供統計數據，不做主觀評論
- 如需說明，註明「根據主計總處定義」
- 不提供投資建議或政策建議

### 4. 錯誤處理

- API 失敗重試 1 次（間隔 2 秒）
- 重試後仍失敗則回報錯誤
- 記錄錯誤日誌供除錯

### 5. 效能考量

- 快取常用資料表的元資料（24 小時）
- 快取功能代碼對照表（每週更新）

## 🔍 Edge Cases

### 1. 模糊查詢處理

**情境**：使用者輸入「CPI」、「物價」等簡稱

**處理**：

- 使用關鍵字匹配找出可能的資料表
- 列出前 3 個最相關的選項讓使用者選擇
- 提供各選項的簡短說明

### 2. 跨年度缺漏數據

**情境**：某些年度數據尚未公布或歷史數據缺漏

**處理**：

- 回傳有數據的年份
- 明確標註缺漏年份
- 說明可能原因（如：資料尚未公布、歷史資料未數位化）

### 3. 季度/月度轉換

**情境**：使用者要年度數據，但資料表僅提供月度

**處理**：

- 說明此表僅提供月度數據
- 建議查詢所有月份後自行計算年度平均/總和
- 或推薦其他提供年度數據的資料表

### 4. 資料表改版

**情境**：同一統計項目有多版本（如：行業分類第 8、9、10 次修訂）

**處理**：

- 優先使用最新版本
- 標註版本資訊
- 如查詢歷史年度，自動使用該時期的適用版本

### 5. 多語言維度名稱

**情境**：使用者用英文查詢（如「employment」）

**處理**：

- 建立中英對照表（常見統計詞彙）
- 翻譯後再進行查詢
- 輸出時可選擇雙語呈現

### 6. 即時數據查詢

**情境**：查詢當月或當季數據（可能尚未公布）

**處理**：

- 檢查資料表最新可用時間
- 提示：「2026 年 2 月數據預計於 2026 年 3 月底公布」
- 提供最新已公布的數據

## 🆘 常見問題（FAQ）

### Q1: 為什麼查不到最新一期的資料？

**A**: 主計總處統計資料有公布時程。例如：

- 月資料：通常次月底公布
- 季資料：季後 1-2 個月公布
- 年資料：次年 2-3 月公布完整數據

### Q2: 如何知道某個資料表有哪些維度可以篩選？

**A**: 使用「查看 [資料表名稱] 的維度結構」指令，系統會列出所有可用維度和項目。

### Q3: 可以一次查詢多個統計表嗎？

**A**: 可以，使用批次查詢功能（進階功能）。

### Q4: 數據的單位是什麼？

**A**: 系統會自動標註單位（如：千人、百萬元、%、指數等），依各資料表定義。

### Q5: 歷史資料可以查到多久以前？

**A**: 依資料表而異，大多數可查詢至民國 67 年（1978 年）以後，部分更早。
